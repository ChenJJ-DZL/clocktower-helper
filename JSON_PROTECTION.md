# JSON 文件夹保护协议

## 重要通知
本项目中的 `/json` 文件夹包含所有游戏核心数据文件，这些文件已经过完整规范化处理，格式统一且经过验证。

## 核心规则
**未经用户明确授权，禁止修改 `/json` 文件夹内的任何文件。**

## 为什么需要这个规则？
1. **数据完整性** - JSON文件已经过精心整理，格式统一（参照小恶魔示例）
2. **测试稳定性** - 项目测试依赖于这些数据的稳定结构
3. **Token节约** - 不必要的修改会导致大量验证工作，浪费token
4. **项目健康** - 保持数据层的纯净有助于长期维护

## 具体保护范围
- `json/full/` 所有角色JSON文件
- `json/` 目录下的任何JSON、TXT、MD文件
- 任何游戏数据文件

## 操作流程
当任务涉及JSON文件夹时，AI助手必须：

1. **检查授权** - 查看 `json/.clinerules` 文件
2. **确认指令** - 确保用户明确授权修改特定文件
3. **最小化修改** - 只修改明确指定的部分
4. **验证结果** - 修改后运行 `npm run type` 和 `npm run test`

## 例外情况
只有在以下明确情况下可以修改：
- 用户明确指示修改特定角色
- 用户授权批量格式转换
- 修复外部工具引入的语法错误
- 用户要求添加新角色/剧本

## 违规后果
未经授权修改将导致：
- 游戏数据损坏
- 类型检查失败
- 测试用例破坏
- 需要大量修复工作
