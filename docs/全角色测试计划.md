# 血染钟楼 全角色测试计划

> 目标：每个角色的技能触发流程(E2E) + 技能效果(Unit) 双重验证
> 排除：旅行者(5)、传奇角色(10) — 待代码实现后补充
> 总角色数：约 120 个（三大剧本 71 + 实验角色 34 + 其他 15）

---

## 一、测试分层策略

```
┌─────────────────────────────────────────────────────┐
│                    E2E 测试层                         │
│  验证：技能触发流程正确（叫醒→选目标→确认→反馈）      │
│  工具：Playwright                                     │
│  粒度：每个交互型角色 1 个 E2E 测试                    │
├─────────────────────────────────────────────────────┤
│                   单元/仿真测试层                      │
│  验证：技能效果正确（状态变更、信息生成、胜负判定）     │
│  工具：Vitest                                         │
│  粒度：每个能力文件 1-3 个测试用例                      │
├─────────────────────────────────────────────────────┤
│                   花名册测试层                         │
│  验证：角色可分配、游戏可启动、首夜可推进              │
│  工具：Playwright (roster_fixed.spec.ts)               │
│  粒度：覆盖所有角色（含被动/信息类）                   │
└─────────────────────────────────────────────────────┘
```

npm run check:all && npm run test:e2e

npx playwright test tests/roster_fixed.spec.ts
