# 规则重新采集计划

## 问题描述
`json/rule`文件夹中的规则文件内容损坏，显示"[内容解析错误，请查看原始文件]"，需要重新采集。

## 目标
1. 删除损坏的规则文件
2. 从官方Wiki重新采集规则内容
3. 使用备份文件格式保存（包含来源、采集时间、标题、内容字段）

## 备份文件格式分析
```json
{
  "来源": "https://clocktower-wiki.gstonegames.com/index.php?title=%E9%A6%96%E9%A1%B5",
  "采集时间": "2026-03-28 20:43:19",
  "标题": "规则标题",
  "内容": "规则内容..."
}
```

对于夜晚行动顺序文件：
```json
{
  "来源": "https://clocktower-wiki.gstonegames.com/index.php?title=%E9%A6%96%E9%A1%B5",
  "采集时间": "2026-03-28 20:43:19",
  "标题": "夜晚行动顺序一览（首夜）",
  "说明": "黄昏阶段为旅行者行动时间，后续按角色类型顺序唤醒",
  "行动顺序": [
    {
      "序号": "1.黄昏（旅行者）",
      "描述": "检查所有玩家是否闭上眼睛..."
    }
  ]
}
```

## 需要采集的章节
1. 规则概要
2. 重要细节
3. 术语汇总
4. 给说书人的建议
5. 规则解释
6. 相克规则
7. 可以但不建议
8. 隐性规则
9. 规则调整提前公示
10. 夜晚行动顺序一览（首夜）
11. 夜晚行动顺序一览（其他夜晚）

## 实施步骤
1. 切换到code模式
2. 删除损坏的规则文件
3. 修改`collect_rules.py`脚本以生成所需格式
4. 执行采集脚本
5. 验证采集结果
6. 应用格式修复（如果需要）
7. 最终验证
