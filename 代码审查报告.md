# 代码审查报告 (2026-03-13)

## 2. 架构合理性分析

### 2.1 "上帝 Hook" (God Hook) 碎片化风险
- **现状分析**：`useGameController.ts` 正在从一个数千行的单体逻辑向 `useGameState`、`useGameActions` 等子 Hook 迁移。
- **不合理之处**：
  - **引用断层**：大量变量（如 `gamePhase`, `seats`）在 `useGameController` 中被删除但在下游逻辑中仍被引用，导致千余个 `Name not found` 错误。
  - **重构不彻底**：部分逻辑（如夜晚结算、Gossip 逻辑）在重构过程中出现了**双重定义**或**遗漏**。

### 2.2 夜间逻辑协议不一致 (Protocol Mismatch)
- **现状分析**：`nightLogic.ts` 负责调度所有的角色技能。
- **不合理之处**：
  - **接口滞后**：`NightActionContext` 接口定义的字段与实际角色（如 `Dreamer`, `Clockmaker`）需要的辅助函数（如 `getRegistration`, `getMisinformation`）不匹配。
  - **返回结构混乱**：部分角色返回 `speak` 字段，部分返回 `modal`，缺乏统一的行动结果（Action Result）映射机制。

## 3. 代码问题与冗余检查

### 3.1 冗余代码 (Redundant Code)
- **重复的 Effect**：在 `useGameController.ts` 原 line 1096-1168 处发现了一段废弃的夜晚转换逻辑，这原本是旧版代码的残留，现已清理。
- **副作用冗余**：存在多个 `useEffect` 监听完全相同的依赖项，增加了渲染负担。

### 3.2 错误代码 (Erroneous Code)
- **非法嵌套 Hook**：修复了在 `setCurrentModal` 闭包中错误嵌套 `useEffect` 的严重语法错误（导致页面白屏）。
- **类型滥用**：大量使用 `any` 作为临时补丁（如 `NightActionContext` 中的 `actionData`），导致重构时无法及时发现字段变更带来的风险。
- **无效注释**：代码中存在大量被注释掉的、未翻译的中文提示，建议清理或转入专门的多语言(i18n)资源文件。

---

## 4. 运行 Bug 检查

1. **角色技能失效**：由于 `NightActionContext` 的字段偏差，筑梦师 (Dreamer) 等角色的 `handler` 在执行时会因为解构不到 `isVortoxWorld` 而导致判断逻辑失效（默认 false）。
2. **夜间流程卡死**：如果说书人手动触发了一个未定义的 ModalType，全局模态管理器会导致应用状态死锁。
3. **状态同步延迟**：重构后的 `GameContext` 在高频更新（如多个角色结算）时可能存在闭包陷阱（Stale Closure）。

---

## 5. 后续行动方案 (Action Plan)

1. **[P0] 补齐类型声明**：更新 `roleDefinition.ts` 中的 `NightActionContext` 接口，正式包含 `getRegistration`, `vortoxWorld` 等重度使用的字段。
2. **[P1] 完成变量对齐**：在 `useGameController.ts` 中统一通过 `useGameState` 获取核心状态，彻底消除 1000+ 个 `Name not found` 错误。
3. **[P2] 逻辑去重**：建立统一的 `GameEventProcessor`，将散落在各处的 `useEffect` 状态机逻辑合并，避免 Gossip、中毒、杀戮逻辑重复触发。
4. **[P3] 环境优化**：启用 ESLint 自动检查嵌套 Hook 和无效依赖。
