# 血染钟楼助手 (Clocktower Helper) 全面代码审查报告 v4

## 1. 架构审查 (Architecture Review)

### 1.1 核心组件膨胀 (Hotspot Alert)
- **发现**: `src/hooks/useGameController.ts` 文件体积进一步膨胀至 **2857 行**。
- **评价**: 该文件承载了过多的业务状态、UI 控制逻辑和规则推演，由于文件过大，IDE 感知和维护成本极高。
- **建议**: **最高优先级拆分**。应按功能模块（如 `useVillageState`, `useNightCycle`, `useVoteSystem`）进行水平拆分，或将纯规则逻辑抽离至独立的 `engine/` 目录。

### 1.2 角色系统演进现状
- **发现**: 系统处于从“数据驱动旧模式”向“OOP 角色定义模式”转年的深水区。
- **冗余点**: `app/data.ts` (2093 行) 中保留了大量已标记为 `@deprecated` 的字段（如 `firstNightOrder`, `nightActionType` 等），但仍有存量逻辑依赖它们。
- **进度评估**: 经检查，`src/roles` 中仍有 **29 个角色** 的 `handler` 标记为 `undefined` (TODO: Migrate to OOP)。这导致了逻辑实现的不一致性。

## 3. 错误与 Bug 检查 (Errors & Bugs)

### 3.1 核心逻辑错误 (Critical Errors)
经静态分析，发现以下 **6 处** 导致构建失败的严重错误：

1. **对话框参数缺失** (`src/hooks/useRoleAction.ts:315`):
   - `actionConfig.dialog` 现在强制要求传入 `context` (NightActionContext)，但目前代码仅传入了 2 个参数。
2. **Pacifist 属性不匹配** (`src/components/game/GameModals.tsx:241`):
   - `PacifistConfirmModal` 传入了不存在的 `onSave` 和 `onDoNotSave` 属性，导致组件调用失败。
3. **Execution 结果类型缺失** (`src/roles/outsider/moonchild.ts:81`):
   - `onExecution` 返回的 Modal 数据中缺少必填的 `onResolve` 回调函数。
4. **单元测试签名失效** (`tests/architecture/role_consistency.test.ts`):
   - `calculateNightInfo` 函数签名更新（新增 `nightCount` 参数），但测试用例未同步更新，导致测试无法运行。
5. **模拟脚本类型错误** (`tests/comprehensive_simulation.ts:172`):
   - 传入了 `undefined` 给期望 `number` 类型的参数。

### 3.2 潜在逻辑 Bug
- **UI 状态覆盖**: `SeatNode.tsx` 中仍存在复杂的即席条件判断逻辑（如处理中毒/醉酒的显示逻辑），建议将这些视图状态逻辑下沉到角色 definition 中以保证 SSOT (Single Source of Truth)。

## 4. 冗余代码 (Redundant Code)

- **`app/data.ts`**: 大量旧的角色元数据与 `src/data/rolesData.json`存在重复定义。
- **`jinxUtils.ts`**: 包含大量硬编码的相克规则，建议与 `josn/相克规则.json` 保持同步并自动加载。

## 5. 综合结论

**项目状态: 警告 (WARNING)**

1. **构建中断**: 类型系统的不一致性导致无法进行正式部署。
2. **架构债**: `useGameController` 的膨胀已到达临界点。
3. **迁移不完整**: OOP 角色迁移进度约 60%，存在“两个系统、两套逻辑”的风险。
