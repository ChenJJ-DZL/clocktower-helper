# 代码审查报告 - v5

## 1. 当前状态概览
项目正处于从 legacy 架构向 OOP/RoleDefinition 架构迁移的核心阶段。目前虽然完成了 `useGameController.ts` 的解耦，但由于类型定义不统一和遗留代码未清理，导致构建失败且存在多处逻辑隐患。

### 构建状态：❌ 失败
- **TypeScript 错误**：共 21 处，分布在 6 个文件中。
- **主要原因**：`Role` 类型定义与 `RoleDefinition` 不匹配， legacy 组件试图访问已移除或移动的属性（如 `firstNightOrder`, `otherNight`）。

---

## 2. 核心问题分析

### 2.1 类型系统冲突 (Critical)
`app/data.ts` 中的 `Role` 接口目前不包含 `firstNightOrder`、`otherNightOrder`、`firstNight`、`otherNight` 等字段。
- **影响文件**：`BarberSwapModal.tsx` (理发师交换)、`RoleInfoModal.tsx` (角色信息预览)、`gameRules.ts` 等。
- **现象**：这些文件直接访问 `role.firstNightOrder` 导致编译错误。

### 2.2 角色能力迁移不完整
仍有约 22 个角色的 `handler` 为 `undefined`（通过 `grep` 确认）。
- **风险**：这些角色在夜晚行动时无法正确触发自动化逻辑，依赖于 legacy 的 `switch/case`。

### 2.3 数据冗余与不一致
- `app/data.ts` 中硬编码了大量的角色属性。
- `src/data/rolesData.json` 包含重复的元数据。
- `src/roles/` 目录下的具体角色定义由于包含 `detailedDescription`，使得 `app/data.ts` 中的 `ability` 字段变得冗余。

---

## 3. 待修复建议

### 3.1 稳定构建 (High Priority)
1.  **统一类型定义**：更新 `app/data.ts` 中的 `Role` 接口，将 `firstNightOrder`, `otherNightOrder`, `firstNight`, `otherNight` 设为可选字段，或者在访问处统一采用 `(role as any)` 桥接。
2.  **修复 Modal 报错**：修复 `BarberSwapModal` 和 `RoleInfoModal` 中对缺失属性的访问。
3.  **修复 logicDispatcher 报错**：修复 `npx tsc` 扫出的 21 处具体报错。

### 3.2 架构清理 (Medium Priority)
1.  **清理 `app/data.ts`**：移除已经迁移至 `rolesData.json` 的重复元数据。
2.  **Jinx 加载优化**：确保 `jinxUtils.ts` 能够通过 `jinxes.json` 动态加载，而不是在各处硬编码判定逻辑。

### 3.3 逻辑补完
1.  **完成余下 22 个角色的 Handler 实现**。
2.  **验证模拟部署**：确保 `npm run build` 和 `npm run start` 能够完整跑通。

---

## 4. 结论
目前项目的重构已完成 80%，重点在于“收尾”和“兼容性处理”。通过修复 21 个编译报错并清理冗余数据，项目将达到可稳定运行、可顺利部署的状态。
