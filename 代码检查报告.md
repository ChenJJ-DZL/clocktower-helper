# 血染钟楼助手 (Clocktower Helper) 代码检查报告

## 1. 项目架构 (Project Architecture)
本项目是一个基于 **Next.js 15** 和 **React 19** 的 Web 应用程序，主要用于辅助《血染钟楼》(Blood on the Clocktower) 游戏的进行。项目的主要架构和技术选型如下：
- **核心框架**: Next.js 15.1.6 (App Router 模式)
- **UI & 样式**: TailwindCSS 4, Framer Motion (用于动画和过渡)
- **状态管理**: 借助 React Hooks (主要使用了 `useGameController`, `useGameState` 等自定义 Hooks 集中管理游戏状态)。
- **测试框架**: 单元测试和集成测试使用 `Jest` (`ts-jest`)，端到端测试与浏览器自动化使用 `Playwright`。
- **语言和类型**: 采用 TypeScript，保证了强类型安全和更好的开发体验。

项目目录结构清晰：
- `src/components`: 拆分细致的可见 React UI 组件（基于新一轮重构拆分了 `GameModals` 等）。
- `src/roles`: 游戏内各角色（镇民、外来者、爪牙、恶魔等）的具体技能与逻辑实现。采用了面向对象的方法 (OOP) 定义各个角色的能力，并与游戏状态解耦。
- `src/hooks`: 核心逻辑交互层，封装核心业务如白天夜晚交替、角色行动队列 (`useNightActionHandler`)。
- `src/contexts`: React 上下文，提供全局状态。
- `src/utils`: 通用的工具函数。
- `tests/`: 包含针对核心逻辑、夜晚唤醒队列、角色一致性等内容的单元测试 (`Jest`) 以及一些端到端/全量游戏模拟测试。
- `josn/`: 存放了包含角色数据、规则、相克说明的大量静态 JSON 数据源文件。

## 2. 代码逻辑 (Code Logic)
- **状态分离与纯函数化**: 最近的重构显著改善了代码逻辑，将表现层的 UI 弹窗状态 (`showXxxModal`) 和游戏的核心数据状态梳理整合在 `currentModal` 集中管理，清除了冗余。核心事件处理 (`processGameEvent`) 运用了纯逻辑方法推演游戏胜负计算，防止了状态泄露问题。
- **角色动作抽象化**: 角色的能力（尤其是夜晚行动）已经被高度提炼与模块化，如占星师、送葬者等的逻辑独立维护在 `src/roles/` 目录下，与流程控制（夜晚醒来队列、确认对话框回调）有较好的衔接设计。
- **队列与闭包问题修复**: 夜晚唤醒的索引 `currentWakeIndex` 通过引入 `useRef` 做引用保持，解决了 React 闭包导致获取到陈旧状态的常见 Bug，逻辑的鲁棒性由此大幅提升。

**总结**: 本项目在代码质量、架构规范以及测试自动化的层面上具有很高的标准，当前版本逻辑健康，可以直接安全地发布和生产部署。
