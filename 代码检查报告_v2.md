# 血染钟楼助手 (Clocktower Helper) 代码检查报告 v2

## 1. 项目概况与架构 (Project Overview & Architecture)
本项目是一个功能完备的《血染钟楼》移动辅助工具，基于现代 Web 技术栈构建。

- **核心技术栈**: 
  - **Next.js 15.1.6 (App Router)**: 提供高效的路由和 SSR/SSG 支持。
  - **React 19.2.0**: 使用最新的 React 特性，包括改进的 Hook 和并发渲染支持。
  - **TailwindCSS 4**: 最新的 Utility-first CSS 框架，极大简化了响应式 UI 开发。
  - **Framer Motion**: 提供平滑的游戏动画过渡，提升用户体验。
  - **TypeScript**: 全量类型定义，显著降低了逻辑复杂度导致的 Bug 率。

- **架构设计**:
  - **OOP 角色系统**: 通过 `src/roles` 目录下的角色定义类（如 `RoleDefinition`），实现了游戏逻辑与 UI 的解耦。角色能力（夜晚、白天、处决）均模块化映射。
  - **集中化状态管理**: 核心状态由 `src/hooks/useGameState.ts` 管理，业务逻辑通过 `useGameController.ts` 及一系列子 Hook（如 `useDayActions`, `useNightActionHandler`）进行横向切分。
  - **上下文通信**: 利用 `GameContext` 和 `GameActionsContext` 实现了全局状态和操作函数的下发，减少了 Prop Drilling。

## 2. 代码逻辑与质量审查 (Code Logic & Quality)

- **核心逻辑鲁棒性**:
  - 引入了 `useRef` 来管理 `currentWakeIndex` 和 `killPlayer` 实现，有效规避了 React 闭包导致的过时状态（Stale Closure）问题。
  - 角色能力触发流程清晰：提名 -> 投票 -> 处决 -> `onExecution` 回调 -> 状态更新 -> 胜负检测。
  - **正在进行的改进**: 正在将直接操作 `Seat` 状态的动作（如修补匠猝死）统一接入 `killPlayer` 接口，以确保所有死亡副作用（如红隐士、恶魔传承等）能被正确触发。

- **代码规范**:
  - 文件命名规范，业务逻辑抽象程度高。
  - 部分 Hook（如 `useGameController`）体积较大（>2800行），建议未来进一步拆分为更细粒度的逻辑单元以提升可测试性。

## 3. 运行与测试状态 (Operation & Testing)

- **测试体系**:
  - **Jest**: 负责核心逻辑（`logic_core.test.ts`）和角色一致性的单元测试。
  - **Playwright**: 负责高层 E2E 模拟，涵盖了从首日设置到复杂扩充角色（如 `Tinker`, `Moonchild`）的完整流程。
- **验证结果**:
  - 基础逻辑测试通过。
  - 扩充角色 E2E 测试正处于单 worker 调试阶段，旨在消除多线程运行导致的竞态冲突和挂起问题。

## 4. 部署与环境 (Deployment & Environment)

- **部署就绪性**:
  - 项目配置了标准的 `npm run build` 和 `npm run start` 流程。
  - 官方在线预览地址为 `baigangroup.fun`，部署于 Vercel 平台，支持自动化部署。
- **环境兼容性**:
  - 支持最新的 Node.js 环境。
  - 通过 `isPortrait` 状态实现了良好的移动端适配，适合说书人手持平板/手机操作。

## 5. 改进建议 (Recommendations)

1. **逻辑解耦**: 继续推进 `DayActionContext` 的完善，将残留的手动状态修改逻辑（如手动设置 `isDead: true`）统一替换为 `killPlayer` 调用。
2. **测试加速**: E2E 测试目前包含大量 `sleep` 调用，建议通过 Playwright 的 `expect` 自动重试机制替换固定等待，以缩短测试时间。
3. **复杂角色文档**: 对于 `Zombuul` 等具有多重生命逻辑的角色，建议在代码注释中增加具体的状态变迁图示。
