## 第一批 E2E 测试已知 SKIP（2026-06-21）

### SKIP-01: 修补匠(BMR) 白天按钮被遮罩层覆盖 (Intermittent)
- **现象**: BMR 剧本中，白天按钮被 `z-50 bg-black/60` 模态遮罩层阻挡，Playwright 无法点击
- **根因**: BMR 剧本的夜间说书人指导弹窗在进入白天时未完全关闭，残留遮罩层覆盖了UI元素
- **涉及文件**: `tests/night_helper.ts` — `skipToDay()` 函数
- **修复路径**: 在白天检测前增加"强制关闭所有残留遮罩"步骤，遍历并关闭所有 `data-modal-key` / `.fixed.inset-0.z-50` 元素
- **状态**: ⏳ 待修复

### SKIP-02: 占卜师(TB) 结果弹窗需点"是/否"而非任意按钮
- **现象**: 占卜师首夜选2目标后弹出结果弹窗，通用 night_helper 点击了错误按钮导致游戏状态未推进
- **根因**: `night_helper.ts` 的模态弹窗处理器对所有弹窗都点击第一个可用按钮，但占卜师结果弹窗需要特定点击"是"或"否"按钮
- **涉及文件**: `tests/night_helper.ts` — 模态弹窗处理逻辑
- **修复路径**: 识别 `data-modal-key` 包含 "占卜师 - 结果" 的弹窗，专门点击"是"按钮，然后等待确认按钮
- **状态**: ⏳ 待修复

### 🛠️ 修复记录：新引擎执行路径桥接（2026-06-02）

**问题**：尽管角色能力已 100% 迁移到新引擎（98 个 `.ability.ts`），但 `useNightActionHandler.handleNightAction()` 仍在查找旧角色定义文件（`roleRegistry`）中的 `handler`。由于旧 handler 已被断开，大量角色返回 `false`，导致游戏流程卡住、技能无法交互。

**修复方案**：在 `handleNightAction` 中增加桥接路径：

1. 当旧 handler 不存在时，从 `getRawAbilityMap()` 获取新引擎能力
2. 构建 `MiddlewareContext`，调用 `runFullAbilityPipeline()` 执行能力
3. 从返回的 snapshot 中提取更新的座位状态，同步回 React 状态
4. 处理日志、弹窗、队列推进等副作用

**涉及文件**：
- `src/hooks/useNightActionHandler.ts` — 新增 `executeViaNewEngine()` 桥接函数
- `src/utils/abilityExecutor.ts` — 支持异步 handler（`Promise<boolean>`）
- `src/hooks/useExecutionHandlers.ts` — `confirmPoison`/`confirmPoisonEvil` 适配异步
- `src/hooks/useGameController.ts` — `handleConfirmActionImpl` 适配异步
- `src/hooks/useInteractionHandler.ts` — `handleConfirmAction` 适配异步

**清理工作**：
- 删除无外部导入的 `src/hooks/useNightLogic.ts`
- 移除 `useNightActionHandler.ts` 中未使用的 `RoleDefinition` 导入

**测试覆盖**：
- 新增 `tests/new_engine_bridge.test.ts` — 6 个测试用例
- 验证 98 个能力全部注册、中间件数组完整、核心角色可执行、死亡被中止
- 全部 40 个已有测试通过 ✅

### 🗃️ rolesData.json meta 数据补齐（2026-06-02）

**问题**：98 个有 `.ability.ts` 的角色中，42 个在 `src/data/rolesData.json` 中缺失 `firstNightMeta`/`otherNightMeta` 等 UI 交互数据，导致说书人界面无法生成正确的唤醒提示和目标选择配置。

**修复方案**：

| 批次 | 范围 | 数量 |
|:---|---:|:---:|
| 第 1 批 | 在 `rolesData.json` 中但缺 meta 的角色 | 15 个 |
| 第 2 批 | 完全不在 `rolesData.json` 中的角色（新增条目） | 27 个 |
| 无需处理 | 被动角色（`PASSIVE` 触发类型，无夜间行动） | 2 个 |

**数据来源**：
- 从 `src/roles/new_engine/*.ability.ts` 自动提取 `targetConfig`（min/max/allowSelf/allowDead）
- 从 `triggerTiming` 确定 `firstNight`/`otherNight` 归属
- 从 JSDoc 和 `abilityName` 生成中文提示词（script/instruction）

**影响**：
- `rolesData.json` 角色数: 171 → **198**
- 有 ability 且不在 rolesData 中的角色: 27 → **0**
- 在 rolesData 中缺 meta 的角色: 15 → **2**（recluse/tinker 为 PASSIVE 角色，符合设计）
- `npm run test:simulation:tb` — 5/5 通过 ✅
