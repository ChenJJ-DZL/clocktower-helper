# 血染钟楼项目与官方JSON文档未对齐映射报告

## 统计信息
- 官方角色总数：超过150个（含全扩展）
- 项目已实现角色：98个（镇民58+外来者22+爪牙25+恶魔19）
- 缺失角色：50+（主要为国风扩展：华灯初上、山雨欲来等）

---

## 一、角色能力框架层未对齐

### 1.1 能力系统架构缺失
| 官方能力类别 | 项目实现状态 | 影响范围 |
|-------------|-------------|---------|
| 回溯型能力 | 部分实现，无统一框架 | 送葬者、造谣者、数学家等20+角色 |
| 持续检测型能力 | 完全未实现统一框架 | 占卜师、茶艺师、哲学家等15+角色 |
| 互动干扰能力 | 部分硬编码实现，无通用机制 | 陌客、间谍、僵怖等10+角色 |
| 认知覆盖能力 | 部分实现，无统一处理逻辑 | 酒鬼、疯子、提线木偶等5+角色 |
| 死后能力保留 | 部分实现，无统一校验 | 陌客、间谍、亡骨魔等8+角色 |
| 特殊胜利条件 | 零散实现，无统一优先级判断 | 主谋、镜像双子、圣徒等10+角色 |

### 1.2 核心机制不符合官方规则
| 官方规则 | 项目实现问题 |
|---------|-------------|
| 醉酒/中毒机制：仅影响能力结果，不影响触发时机 | 项目中醉酒/中毒会直接阻止能力触发 |
| 限次能力：醉酒使用仍消耗次数 | 项目中使用失败不消耗次数 |
| 角色变化：获得新角色后重置所有限次能力 | 项目中限次次数全局共享 |
| 邻近判定：跳过不满足条件的玩家取最近符合条件的 | 项目中仅计算左右相邻2名，不支持条件过滤 |
| 处决结算优先级：先判断游戏结束再结算其他能力 | 项目中结算顺序混乱，可能导致提前触发死亡能力 |

## 三、游戏规则层未对齐

### 3.1 游戏流程规则
| 官方规则 | 项目实现问题 |
|---------|-------------|
| 白天处决后立即进入夜晚，每天最多1次正常处决 | 支持多次处决，无阶段跳转逻辑 |
| 提名阶段：每名玩家每天只能发起1次提名，被提名1次 | 无提名次数限制 |
| 投票规则：死亡玩家仅有1次投票机会 | 死亡玩家可无限投票 |
| 夜晚行动顺序：严格按照官方顺序执行，中途角色变化需插入结算 | 固定顺序，无动态插入机制 |

### 3.2 能力交互规则
| 官方规则 | 项目实现问题 |
|---------|-------------|
| 能力生效优先级：咖啡师>酿酒师>涡流>醉酒/中毒>自身能力限制 | 无统一优先级判断，按编码顺序执行 |
| 互动干扰规则：同一角色在不同能力判定中可返回不同结果 | 固定返回同一结果 |
| 回溯型能力：仅在生效节点判断是否醉酒/中毒 | 按触发节点状态判断 |
| 更换选择目标：仅影响玩家选择，不影响说书人决定的效果 | 完全未实现该机制 |
