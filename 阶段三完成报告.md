### 1. 统一的夜晚行动顺序机制
**文件位置**: `src/utils/unifiedNightOrder.ts`
**功能特性**:
- 整合了 `nightQueueGenerator`、`nightOrderParser`、`nightOrderOverrides` 等组件
- 提供统一的API接口，支持多种顺序来源（官方顺序、角色定义顺序、覆盖顺序）
- 支持夜晚顺序验证和预览功能
- 提供详细的顺序信息，包括唤醒条件和来源

**核心接口**:
```typescript
interface UnifiedNightOrderConfig {
    isFirstNight: boolean;
    enableOfficialOrder: boolean;
    enableRoleDefinitionOrder: boolean;
    enableOverrideOrder: boolean;
}

interface NightOrderResult {
    sortedSeats: Seat[];
    orderDetails: Array<{
        seatId: number;
        roleId: string;
        roleName: string;
        order: number;
        source: "official" | "roleDefinition" | "override" | "fallback";
        wakeCondition?: string;
    }>;
    totalItems: number;
    isFirstNight: boolean;
}
```

### 2. 统一的角色能力定义机制
**文件位置**: `src/roles/unifiedRoleDefinition.ts`
**功能特性**:
- 整合了 `roleAbility.types.ts`、角色定义系统等组件
- 提供统一的能力定义、注册和执行接口
- 支持能力的前置条件检查、执行和后置处理
- 与游戏事件总线集成，发布能力相关事件

**核心接口**:
```typescript
interface UnifiedAbilityConfig {
    roleId: string;
    abilityId: string;
    abilityName: string;
    triggerTiming: AbilityTriggerTiming[];
    wakePriority: number;
    firstNightOnly: boolean;
    wakePromptId: string;
    targetConfig: {
        min: number;
        max: number;
        allowSelf: boolean;
        allowDead: boolean;
    };
    preCondition?: (seat: Seat, context: AbilityContext) => boolean;
    execute: (context: AbilityContext) => Promise<AbilityResult>;
    postProcess?: (result: AbilityResult, context: AbilityContext) => void;
}
```

### 3. 统一的游戏事件总线机制
**文件位置**: `src/utils/unifiedEventBus.ts`
**功能特性**:
- 整合了 `gameEventBus` 并添加更多高级功能
- 提供统一的事件发布、订阅、过滤和监控接口
- 支持事件历史记录和性能监控
- 提供事件等待、批量发布等高级功能

**核心接口**:
```typescript
interface EventListenerConfig<T extends GameEventType> {
    listenerId: string;
    eventType: T;
    callback: (payload: GameEventMap[T]) => void;
    priority?: number;
    once?: boolean;
    filter?: (payload: GameEventMap[T]) => boolean;
    context?: any;
}

interface EventMonitorConfig {
    enableHistory: boolean;
    maxHistoryLength: number;
    enableDebug: boolean;
    enablePerformance: boolean;
}
```

## 技术改进

### TypeScript兼容性修复
1. **添加了 `downlevelIteration` 配置**: 在 `tsconfig.json` 中添加了 `"downlevelIteration": true` 以支持ES5环境下的Map迭代
2. **修复了JSON导入问题**: 修改了 `nightOrderParser.ts` 使用动态导入避免TypeScript编译错误
3. **统一了Map迭代方式**: 所有文件都使用 `Array.from()` 来避免downlevelIteration问题
